### 1. 因子数据迁移工具开发 ✅ (2026-04-01)

#### 背景
原 `stock_factors` 表混合存储日频因子和月频因子，数据结构混乱，不利于后续因子计算和分析。

#### 解决方案
创建独立的日频因子表和月频因子表，实现数据规范化存储。

#### 数据库表结构

**stock_daily_factors** (日频因子表):
- 与 `kline_data` 日期对齐
- 27 个因子字段：
  - 市值类：circ_market_cap, total_market_cap, days_since_ipo
  - 市场表现类：change_10d, change_20d, bias_5/10/20, amplitude_5/10/20, change_std_5/10/20, amount_std_5/10/20
  - 技术指标类：kdj_k/d/j, dif, dea, macd
  - 估值类：pe_inverse, pb_inverse
  - 目标变量：next_period_change
  - 标记：is_traded

**stock_monthly_factors** (月频因子表):
- 月末日期为报告期
- 18 个因子字段：
  - 财报时间：report_quarter, report_year
  - 利润类：net_profit, net_profit_ttm, net_profit_ttm_yoy, net_profit_single, net_profit_single_yoy, net_profit_single_qoq
  - 现金流类：operating_cash_flow, operating_cash_flow_ttm, operating_cash_flow_ttm_yoy, operating_cash_flow_single, operating_cash_flow_single_yoy, operating_cash_flow_single_qoq
  - 资产类：net_assets
  - 行业分类：sw_level1, sw_level2, sw_level3

#### 迁移规则
- 只迁移 2021-04-02 之后的数据（与 kline_data 对齐）
- next_period_change 从 kline_data 计算：`(次日收盘价 - 当日收盘价) / 当日收盘价`
- 日频因子在月内向前填充
- 原始 stock_factors 表保持不变

#### 迁移结果
```

  - 总记录数：5,735,525
  - 股票数量：~1,500
  - 日期范围：2021-04-02 至 2026-04-02

  - 总记录数：295,320
  - 股票数量：~1,500
  - 日期范围：2021-04-02 至 2026-04-02
```

#### 解决的问题

**问题 1：Linux 路径兼容性**
- SDK 硬编码 Windows 路径 `D://AmazingData_local_data//`
- 解决方案：使用 `is_local=False` 在线模式调用 API

**问题 2：数据单位转换**
- SDK 返回的股本数据单位：万股
- SDK 返回的财务数据单位：元
- 统一转换为亿元进行计算

**问题 3：SDK API 返回格式**
- 单只股票：返回 DataFrame
- 多只股票：返回 dict（key 为股票代码）

#### 已实现的 API 封装

**sdk_api.py** - SDK API 封装层
```python
class AmazingDataAPI:
    - get_equity_structure()      # 股本结构数据
    - get_income_statement()      # 利润表数据
    - get_cash_flow_statement()   # 现金流量表数据
    - get_balance_sheet()         # 资产负债表数据
    - get_industry_base_info()    # 行业分类基础信息
```

#### 因子计算集成

**daily_factor_calculator.py** 更新
```python
calculate_market_cap():
  - 从 SDK 获取总股本、流通股本
  - 计算流通市值 = 流通股本 × 收盘价 / 10000
  - 计算总市值 = 总股本 × 收盘价 / 10000

calculate_valuation():
  - 从 SDK 获取净利润 TTM、净资产
  - 计算 PE 倒数 = 净利润 TTM / 市值
  - 计算 PB 倒数 = 净资产 / 市值
```

**monthly_factor_calculator.py** 更新
```python
get_financial_data():
  - 从 SDK 获取利润表、现金流量表、资产负债表
  - 自动筛选对应季度数据
  - 计算 TTM 值

get_industry_classification():
  - SDK 暂无单只股票行业分类接口
  - 预留接口，后续可通过反向构建映射实现
```

## 技术说明

### SDK 调用注意事项

1. **登录状态缓存**
   ```python
   _login_cache = None  # 类变量缓存登录状态
   ```

2. **在线模式调用**
   ```python
   # 错误方式（会下载数据到 Windows 路径）
   info.get_equity_structure(["689009.SH"], is_local=True)
   
   # 正确方式（在线查询）
   info.get_equity_structure(["689009.SH"], is_local=False)
   ```

3. **单位转换规则**
   ```python
   # 市值计算（结果：亿元）
   market_cap = shares * close / 10000
   
   # 估值计算（财务数据转换为亿元）
   net_profit_yi = net_profit / 100000000
   pe_inverse = net_profit_yi / market_cap
   ```
