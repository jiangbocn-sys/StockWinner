## 问题修复

### 1. 选股候选数据锁死问题

**问题**：
- 临时候选表 `temp_candidates` 有外键约束指向 `strategies(id)`
- 删除策略时，如果该策略有候选数据，删除失败
- 选股前未清除旧数据，导致新旧策略候选股票混淆

**修复**：
- `services/screening/service.py`: 选股前自动清除该账户的旧候选数据
- `services/ui/strategies.py`: 删除策略前先删除关联的临时数据

```python
# services/screening/service.py
if pending_to_temp:
    await db.execute("DELETE FROM temp_candidates WHERE account_id = ?", (account_id,))

# services/ui/strategies.py
await db.execute("DELETE FROM temp_candidates WHERE strategy_id = ?", (strategy_id,))
```

### 2. 选股进度条不更新问题

**问题**：
- 手动选股时 `screening.running` 始终为 `false`
- 前端轮询逻辑依赖 `screening.running` 判断是否继续
- 导致轮询立即停止，进度条不更新

**修复**：
- `frontend/src/views/Watchlist.vue`: 修改完成判断逻辑
- 改用 `current_phase` 和已处理股票数判断

```javascript
// 修复前
if (progress.current_phase === 'done' || !data.screening?.running) {

// 修复后
const isCompleted = progress.current_phase === 'done' ||
                    (progress.total_stocks > 0 && progress.processed >= progress.total_stocks)
```

### 3. 策略删除失败问题

**问题**：
- 策略被临时候选数据引用时，外键约束阻止删除

**修复**：
- 删除策略前先删除关联的临时候选数据

### 4. 选股服务关闭候选窗口后按钮未恢复

**问题**：
- 关闭候选对话框时只设置了 `showCandidatesDialog = false`
- 未重置 `screeningProgress.processing`
- 导致选股按钮保持禁用状态

**修复**：
- 新增 `cancelCandidates()` 函数，关闭对话框同时重置进度状态

## 架构优化

### 选股程序优化（第一阶段完成）

#### 核心思路

**"因子优先，按需计算"**

1. 优先使用 `stock_daily_factors` 表中的预计算因子
2. 只对数据库中不包含的指标进行动态计算
3. 新增选股条件只需丰富因子库，无需修改选股程序

#### 新增文件

1. **`services/screening/factor_registry.py`** (因子注册表)
   - 管理 30+ 个可用因子
   - 条件解析器：从 `"MA5 > MA20"` 提取 `{MA5, MA20}`
   - 因子分类：区分数据库因子和需计算因子
   - 批量查询：一次性获取所有数据库因子

2. **`docs/SCREENING_OPTIMIZATION_PLAN.md`** (优化方案文档)
   - 详细的设计思路
   - 实施步骤
   - 性能对比

#### 修改文件

1. **`services/screening/service.py`**
   - 新增 `_evaluate_conditions_from_local_optimized()` 方法
   - 优先使用优化版本，失败自动回退到传统版本
   - 支持动态计算缺失因子

#### 因子映射

| 因子类别 | 数据库因子 (17 个) | 动态计算因子 (10+ 个) |
|---------|-----------------|---------------------|
| 均线类 | - | MA5, MA10, MA20, MA60 |

| 振幅类 | amplitude_5, amplitude_10, amplitude_20 | - |
| 波动率类 | change_std_5, change_std_10, change_std_20 | - |
| 成交额波动 | amount_std_5, amount_std_10, amount_std_20 | - |
| 估值类 | pe_inverse, pb_inverse | - |

| 其他 | days_since_ipo, next_period_change | - |
| 基础指标 | - | RSI, EMA12, EMA26 |

#### 性能对比

**传统模式**（条件：`MA5 > MA20`, `RSI < 30`）：
- 计算 MA5, MA10, MA20, MA60: 5194 × 4 = 20,776 次
- 计算 RSI: 5194 次
- 计算 KDJ: 5194 次
- 计算 MACD: 5194 次
- **总计**：~36,000 次计算

**优化模式**：
- 数据库读取 MACD, KDJ 等：0 次计算
- 动态计算 MA5, MA20, RSI: 5194 × 3 = 15,582 次
- **总计**：~15,600 次计算
- **减少**：56% 计算量

**预期提升**：
- 计算量减少 50%+
- 选股时间从 2-3 分钟降至 30-60 秒
- 新增选股条件无需修改代码
